  • UDP Messenger

    Decentralized messenger written in Java, uses datagram packets to transmit information in a 'JSON-like' format and can use both distance vector or link state routing to route messages via connected hosts.

  • Remote Logger

    Logging engine written in Node.JS with a RethinkDB datastore. Designed to handle very heavy logging loads with real-time transmission, and relaying to connected clients. Uses socket.io, and has plugins for Java and Node.js.

  • Web Server (closed source, plans to open source)

    I've been working on my web stack for about 3-4 months now, it's based around Node.js.

    At its core is the load balanacing proxy, it allows load balancing across multiple threads or multiple machines and is based on the source ip.

    When the user makes a request, it is checked against a database of micro-services, if the micro-service exists, the request is proxied to the micro-service.

    Throughout the entire system, there is a huge amount of logging and error handling, automated notification system if there are errors

    Modules can also communicate with each other, via IPC if local, or other means if remote.

    The proxy also supports proxying and synchronization of websockets, allowing Socket.IO rooms to run relatively seamlessly.

  • Indigo - High Performance Application Proxy

    The goal of indigo is to completely replace my existing web-stack, allowing easier separation of micro-services, as well as providing a secure platform from the outset.

    Inspired by numerous attempts to abstract away the complexities of adding a new service to my stack, Indigo is designed to allow me to just write code and not worry about anything else.

    Designed to be run behind a reverse Proxy such as NGINX, indigo minimizes the amount of internal TCP traffic by using system level sockets (unix sockets/windows - named pipes).

    Indigo uses very few platform specific features, allowing it to be significantly more maintainable and reduces duplication where possible.
